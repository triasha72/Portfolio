Case Study 02 · Independent build

Atlanta Mobility Resilience Digital Twin

A geospatial mobility simulator focused on how disruptions in a transportation network alter travel time, accessibility, and resilience.

← Back to portfolio
Role
Independent project owner
Core question
How do network disruptions change mobility outcomes across space?
Methods
Network analysis, accessibility metrics, disruption scenarios, resilience interpretation
Tools
Python, geospatial libraries, Jupyter, GitHub
Status
Public GitHub project; add screenshots/maps before sharing widely

Why it matters

Transportation systems fail unevenly. A disruption that looks small in network terms can create major access burdens for specific corridors, neighborhoods, or trip types. That makes resilience a spatial and human question, not only an infrastructure metric.

What I built

  • Structured the project around disruption scenarios rather than static network description.
  • Focused outputs on travel-time change, accessibility loss, and resilience signals.
  • Connected geospatial simulation to everyday mobility consequences and decision-oriented interpretation.

What this shows

This project extends my systems lens from aviation to city-scale transportation. The same modeling instincts apply: define the system, perturb it, quantify the response, and translate the result into a clear story about performance and impact.
